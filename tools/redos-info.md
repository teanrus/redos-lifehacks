# 🔍 redos-info.sh - Сбор информации о системе РЕД ОС

[![Version](https://img.shields.io/badge/version-1.0-green.svg)](https://github.com/teanrus/redos-lifehacks/releases)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](/LICENSE)
[![Platform](https://img.shields.io/badge/platform-RED%20OS%207.3-red.svg)](https://redos.red-soft.ru/)
[![Stars](https://img.shields.io/github/stars/teanrus/redos-lifehacks.svg)](https://github.com/teanrus/redos-lifehacks/stargazers)

Скрипт для автоматического сбора детальной информации о системе **РЕД ОС 7.3** (MUROM). Позволяет быстро получить полную картину состояния системы для диагностики, аудита или отчетности.

## 🚀 Быстрый запуск

```bash
# Автоматический сбор информации (без запросов)
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/redos-info.sh | sudo bash

# С сохранением отчета в файл
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/redos-info.sh | sudo bash -s -- -o system-info.txt
```

## Только сетевая информация
```bash
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/redos-info.sh | sudo bash -s -- -s network
```

## 📋 Что собирает скрипт
| Раздел       | Собираемая информация                                                                                        |
| :----------- | :----------------------------------------------------------------------------------------------------------- |
| ОС           | Версия РЕД ОС, имя хоста, дата/время, uptime, загрузка, активные пользователи                                |
| Ядро         | Версия ядра, архитектура, параметры загрузки, список загруженных модулей                                     |
| Оборудование | Процессор (модель, ядра, производитель), оперативная память, swap                                            |
| Диски        | Структура разделов (lsblk), использование файловых систем (df), SMART статус (опционально)                   |
| Сеть         | Сетевые интерфейсы, маршруты, DNS серверы, слушающие порты, проверка интернета                               |
| Пакеты       | Общее количество пакетов, версии ключевых пакетов, настроенные репозитории                                   |
| Сервисы      | Количество запущенных сервисов, статус критических сервисов (sshd, NetworkManager, firewalld, auditd, crond) |
| Безопасность | Статус SELinux, статус Firewall, последние успешные входы, неудачные попытки входа                           |

## 🛠️ Использование
Синтаксис
```bash
redos-info.sh [опции]
```
Опции
| Опция              | Описание                                                    |
| :----------------- | :---------------------------------------------------------- |
| -h, --help         | Показать справку                                            |
| -i, --interactive  | Принудительно включить интерактивный режим (с запросами)    |
| -o, --output FILE  | Сохранить отчет в указанный файл                            |
| -s, --section SECT | Показать только указанную секцию                            |
| -d, --dir DIR      | Директория для сохранения отчетов (по умолчанию: ./reports) |

Доступные секции
- os — информация об операционной системе
- kernel — информация о ядре
- hardware — информация об оборудовании
- disk — диски и файловые системы
- network — сетевая информация
- packages — установленные пакеты
- services — запущенные сервисы
- security — настройки безопасности
- all — вся информация (по умолчанию)

## 📝 Примеры использования
1. Базовый запуск (автоматическое определение режима)
```bash
wget https://github.com/teanrus/redos-lifehacks/releases/latest/download/redos-info.sh
chmod +x redos-info.sh
sudo ./redos-info.sh
```
- Если скрипт запущен из терминала — работает в интерактивном режиме
- Если выполняется через curl | bash — автоматически переходит в неинтерактивный режим

2. Неинтерактивный режим
```bash
sudo ./redos-info.sh -i
```
- Принудительно включает интерактивный режим (будут задаваться вопросы).

3. Сохранение отчета
```bash
# Сохранить в текущую директорию
sudo ./redos-info.sh -o my-system-info.txt
# Сохранить в указанную директорию
sudo ./redos-info.sh -d /var/log/reports -o system-info.txt
```

4. Получение только сетевой информации
```bash
sudo ./redos-info.sh -s network
```

5. Запуск через curl (без сохранения отчета)
```bash
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/redos-info.sh | sudo bash
```

6. Запуск через curl с сохранением отчета
```bash
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/redos-info.sh | sudo bash -s -- -o /tmp/system-report.txt
```

## 🎯 Особенности
✅ Автоматическое определение режима работы
- При запуске из терминала — интерактивный режим (запрос подтверждения)
- При выполнении через curl | bash — неинтерактивный режим (автоматические ответы)

✅ Безопасное чтение ввода
- Использует /dev/tty для чтения ввода, что корректно работает даже при перенаправлении stdin
- Не зависает при выполнении через pipe

✅ Цветной вывод
- Информация выделяется цветами для удобства восприятия
- При сохранении в файл цвета сохраняются как ANSI-коды

✅ Модульная структура
- Каждая секция собирается отдельной функцией
- Легко модифицировать и расширять

📁 Сохранение отчетов
Отчеты по умолчанию сохраняются в директорию ./reports/ с именем в формате: redos-info-20260327-161141.txt

Структура отчета:
- Заголовок с датой и именем хоста
- Полная информация по всем секциям
- ANSI-коды цветов для сохранения форматирования

## 🔧 Требования
- ОС: РЕД ОС 7.3 (MUROM) или совместимая
- Права: root (sudo) для полного сбора информации
- Утилиты:
  - bash 4.0+
  - systemd (для анализа сервисов)
  - ip, ss (для сетевой информации)
  - rpm, dnf (для информации о пакетах)
- Опционально:
  - smartctl — для проверки SMART статуса дисков
  - bc — для вычислений использования памяти

## 🐛 Устранение неполадок
Проблема: Скрипт не ждет ввода при выполнении через curl | bash
Решение: Это нормальное поведение. Скрипт автоматически определяет, что stdin не является терминалом, и переходит в неинтерактивный режим.

Проблема: Ошибка "bc: command not found"
Решение: Установите пакет bc:
```bash
sudo dnf install bc -y
```

Проблема: Нет информации о SMART
Решение: Установите smartmontools:
```bash
sudo dnf install smartmontools -y
```
