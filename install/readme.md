<details>
  <summary>Установка КриптоПро CSP на РЕД ОС 7.3 (install-cryptopro.sh )</summary>

Скрипт для автоматической установки и настройки **КриптоПро CSP** на операционной системе **РЕД ОС 7.3**. Скрипт автоматически определяет последнюю версию КриптоПро в репозитории GitHub, загружает необходимые файлы, устанавливает зависимости и выполняет настройку для работы с Рутокен.

## 🎯 Что делает скрипт

### 1. **Автоматическое определение последней версии**
- Получает информацию о последнем релизе из репозитория `teanrus/redos-setup` через GitHub API
- При недоступности API запрашивает версию вручную

### 2. **Проверка системы**
- Определяет версию ОС, архитектуру и ядро
- Проверяет наличие необходимых утилит (wget, curl, tar)
- Проверяет архитектуру (требуется x86_64)

### 3. **Управление существующей версией**
- Проверяет наличие установленной версии КриптоПро
- Предлагает удалить старую версию перед установкой

### 4. **Установка зависимостей**
| Пакет                | Назначение                            |
| -------------------- | ------------------------------------- |
| `ifd-rutokens`       | Драйверы для работы с Рутокен         |
| `token-manager`      | Менеджер токенов электронной подписи  |
| `gostcryptogui`      | GUI для работы с ГОСТ-шифрованием     |
| `caja-gostcryptogui` | Интеграция с файловым менеджером Caja |
| `pcsc-lite`          | Поддержка смарт-карт и токенов        |
| `pcsc-lite-ccid`     | Драйвер для USB-токенов               |

### 5. **Загрузка и установка КриптоПро**
- Скачивает архив `kriptopror4.tar.gz` из последнего релиза
- Распаковывает архив (структура: `kriptopror4/` → установочные файлы)
- Автоматически определяет тип установщика:
  - `install_gui.sh` — графический установщик (при наличии X-сервера)
  - `install.sh` — текстовый установщик
  - `*.rpm` — установка через RPM-пакеты

### 6. **Настройка Рутокен**
- Добавляет пользователя в группу `pcscd`
- Проверяет подключенные токены
- Устанавливает утилиты для диагностики (при необходимости)

### 7. **Лицензирование**
- Запрашивает серийный номер лицензии
- Устанавливает лицензию через `cryptcp -license -set`

### 8. **Интеграция с файловым менеджером**
- Настраивает интеграцию ГОСТ-шифрования с Caja
- Создает символическую ссылку на расширение

### 9. **Очистка временных файлов**
- Удаляет загруженные архивы и распакованные файлы (опционально)

## 🚀 Использование

### Быстрый запуск
```bash
curl -sL https://raw.githubusercontent.com/teanrus/redos-lifehacks/main/scripts/install/install-cryptopro.sh | sudo bash
```
### С сохранением файла
```bash
wget https://raw.githubusercontent.com/teanrus/redos-lifehacks/main/scripts/install/install-cryptopro.sh
chmod +x install-cryptopro.sh
sudo ./install-cryptopro.sh
```

## 🔧 Особенности работы
- Автоматическое определение версии
- Скрипт использует GitHub API для получения информации о последнем релизе:

```bash
api_url="https://api.github.com/repos/teanrus/redos-setup/releases/latest"
latest_tag=$(curl -s "$api_url" | grep '"tag_name"' | head -1 | cut -d '"' -f 4)
```
## Структура архива
Скрипт ожидает следующую структуру:

```text
kriptopror4.tar.gz
└── kriptopror4/
    ├── install_gui.sh    # Графический установщик
    ├── install.sh        # Текстовый установщик
    └── ...               # Другие файлы установки
```

## Поддержка различных типов установщиков
- Графический режим — при наличии X-сервера и файла install_gui.sh
- Текстовый режим — при отсутствии графики или файла install.sh
- RPM-пакеты — если не найдены установщики

## Настройка Рутокен
- Автоматическое добавление пользователя в группу pcscd
- Проверка подключенных токенов через rutoken-control или pcsc_scan
- Установка pcsc-tools для диагностики (рекомендуется)

## 🐛 Устранение неполадок
### Проблема: Не удается получить последнюю версию
>Решение: Проверьте интернет-соединение и доступность GitHub:
>
>```bash
>curl -I https://api.github.com/repos/teanrus/redos-setup/releases/latest
>```
### Проблема: Ошибка при установке зависимостей
>Решение: Обновите систему и повторите попытку:
>
>```bash
>dnf update -y
>```
### Проблема: Не определяется Рутокен
>Решение:
>
>Проверьте подключение токена
>
>Перезапустите службу PC/SC:
>
>```bash
>systemctl restart pcscd
>```
>Установите утилиты для диагностики:
>
>```bash
>dnf install pcsc-tools
>pcsc_scan -n
>```
### Проблема: Ошибка при установке лицензии
>Решение: Убедитесь, что серийный номер введен корректно и лицензия актуальна.
