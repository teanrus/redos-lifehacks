# Настройка DNF для быстрой загрузки

**Вот несколько рекомендаций по настройке DNF для ускорения загрузки пакетов в РЕД ОС (которая основана на Red Hat / CentOS и использует DNF).**

## ⚙️ Основные настройки в /etc/dnf/dnf.conf
Откройте файл конфигурации:
```bash
sudo nano /etc/dnf/dnf.conf
```
Добавьте или измените следующие параметры:
1. max_parallel_downloads — параллельная загрузка
- Позволяет загружать несколько пакетов одновременно вместо последовательной загрузки.
```ini
max_parallel_downloads=10
```
- По умолчанию: 3 
- Максимум: 20 
>Рекомендуемое значение: 10–20 в зависимости от скорости интернета 
**Эффект: загрузка 10 пакетов вместо 10 последовательных загрузок значительно сокращает общее время.**
2. fastestmirror — выбор самого быстрого зеркала
- Включает поиск зеркала с наименьшим временем отклика (ping).
```ini
fastestmirror=True
```
- По умолчанию: False 
>⚠️ Важно: fastestmirror выбирает зеркало по ping, а не по реальной пропускной способности. Иногда зеркала, предлагаемые по умолчанию через mirrorlist, уже оптимизированы . Рекомендуется протестировать оба варианта.
3. deltarpm — дельта-обновления
- Загружает только изменения между версиями пакетов, а не весь пакет целиком.
```ini
deltarpm=True
```
По умолчанию: True 
>⚠️ Важно: Экономит трафик, но требует больше CPU и операций ввода-вывода для пересборки пакета локально . При хорошем интернете можно отключить (deltarpm=False), чтобы ускорить процесс за счет отказа от пересборки .
## 🧹 Дополнительные рекомендации
4. keepcache — кэширование пакетов
Сохраняет загруженные .rpm файлы для повторного использования.
```ini
keepcache=True
```
По умолчанию: False 
>Эффект: при повторной установке того же пакета он не будет загружаться заново, что ускоряет работу .
5. installonly_limit — ограничение количества ядер
- Ограничивает количество хранимых версий ядра.
```ini
installonly_limit=3
```
- По умолчанию: 3 
>Эффект: экономит место на диске, особенно важно при частых обновлениях ядра.
6. timeout — таймаут подключения
- Уменьшает время ожидания недоступного репозитория.
```ini
timeout=10
```
- По умолчанию: 30 секунд 
>Эффект: при проблемах с сетью DNF быстрее переключится на следующий источник.

##📋 Пример готовой конфигурации
```ini
[main]
gpgcheck=1
installonly_limit=3
clean_requirements_on_remove=True
best=False
skip_if_unavailable=True

# Ускорение загрузки
max_parallel_downloads=10
fastestmirror=True

# Экономия трафика (или отключите если интернет быстрый)
deltarpm=True

# Сохранение кэша
keepcache=True

# Таймаут
timeout=10
```
## 🧪 Применение настроек
- После редактирования конфигурации выполните:
```bash
# Очистка старого кэша (рекомендуется)
sudo dnf clean all

# Обновление метаданных
sudo dnf makecache

# Проверка скорости
time sudo dnf update --refresh
```
**Сравните время выполнения до и после изменений, чтобы оценить эффект.**

>💡 Дополнительные советы
>
>Объединяйте установку пакетов в одну команду вместо нескольких вызовов dnf install — это сокращает количество обращений к репозиториям .
>
>Периодически очищайте кэш командой sudo dnf clean all, если накопилось много старых данных .
>
>Для очень медленных соединений уменьшите max_parallel_downloads до 2–4, чтобы избежать перегрузки канала .

## 📋 Требования и совместимость

| Параметр | Значение |
|----------|----------|
| **ОС** | РЕД ОС 7.3 / 8.0 |
| **Пакетный менеджер** | DNF |
| **Файл конфигурации** | `/etc/dnf/dnf.conf` |
| **Совместимость** | ✅ РЕД ОС 7.x, ✅ РЕД ОС 8.x |

> [!note]
> В РЕД ОС 7.x (на базе CentOS 7) DNF может быть установлен вручную. В РЕД ОС 8.x DNF — пакетный менеджер по умолчанию с более новыми версиями плагинов.
