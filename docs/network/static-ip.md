## 📚 Лайфхаки по настройке статического IP

### 1. Быстрая настройка через nmcli (NetworkManager)

```bash
# Узнать имя подключения
nmcli connection show

# Установить статический IP
nmcli connection modify "Wired connection 1" \
  ipv4.addresses 192.168.1.100/24 \
  ipv4.gateway 192.168.1.1 \
  ipv4.dns "8.8.8.8 8.8.4.4" \
  ipv4.method manual

# Переподключить
nmcli connection up "Wired connection 1"
```

> **Зачем:** Быстрый способ без редактирования конфигурационных файлов.

---

### 2. Резервное копирование конфигурации перед изменением

```bash
# Сохранить текущие настройки
nmcli connection show "Wired connection 1" > ~/network_backup.txt

# Или скопировать файл конфигурации
cp /etc/NetworkManager/system-connections/*.nmconnection ~/backup/
```

> **Зачем:** Возможность быстрого отката при ошибке в настройках.

---

### 3. Проверка сети перед применением настроек

```bash
# Проверить текущие настройки
ip addr show

# Проверить шлюз
ip route show

# Протестировать DNS перед сохранением
nslookup google.com 8.8.8.8
```

> **Зачем:** Убедитесь, что шлюз и DNS работают до применения постоянных настроек.

---

### 4. Автоматический откат при потере связи

```bash
# Скрипт с таймаутом и откатом
nmcli connection modify "connection" ipv4.addresses "192.168.1.100/24"
nmcli connection up "connection"

# Если через 10 секунд нет сети - откат
sleep 10
ping -c1 8.8.8.8 || nmcli connection reload
```

> **Зачем:** Защита от потери доступа к серверу при неверных настройках.

---

### 5. Настройка через /etc/netplan (если используется)

```yaml
# /etc/netplan/01-netcfg.yaml
network:
  version: 2
  ethernets:
    eth0:
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
```

> **Зачем:** Декларативный способ настройки для систем с netplan.

---

### 6. Добавление нескольких IP на один интерфейс

```bash
nmcli connection modify "connection" \
  +ipv4.addresses "192.168.1.101/24" \
  +ipv4.addresses "192.168.1.102/24"
```

> **Зачем:** Для размещения нескольких сервисов на разных IP.

---

### 7. Полезные команды для диагностики

```bash
# Показать все подключения
nmcli connection show

# Показать активные подключения
nmcli connection show --active

# Перезагрузить NetworkManager
systemctl restart NetworkManager

# Посмотреть логи сети
journalctl -u NetworkManager -f
```

## 🤖 Автоматическая настройка через скрипт

Скрипт для автоматической настройки статического IP-адреса в операционной системе **РЕД ОС** (Red OS).

## 📋 Описание

Скрипт предоставляет интерактивный интерфейс для настройки статического IP:

| № | Компонент | Описание |
|---|-----------|----------|
| 1 | **Выбор подключения** | Выбор сетевого интерфейса из списка доступных |
| 2 | **Просмотр текущих настроек** | Отображение текущей конфигурации сети |
| 3 | **Резервное копирование** | Сохранение текущих настроек для отката |
| 4 | **Настройка IP адреса** | Ввод статического IP с валидацией формата |
| 5 | **Выбор маски подсети** | Быстрый выбор (/24, /16, /8) или своя маска |
| 6 | **Настройка шлюза** | Автоматическое предложение шлюза по умолчанию |
| 7 | **Выбор DNS серверов** | Популярные DNS (Google, Cloudflare, Яндекс) или свои |
| 8 | **Дополнительные IP** | Добавление нескольких адресов на интерфейс |
| 9 | **Проверка подключения** | Тестирование связи после применения настроек |
| 10 | **Скрипт отката** | Автоматическое создание скрипта для возврата к DHCP |

## 📖 Возможности

### ✅ Интерактивный режим
- Выбор подключения из списка всех сетевых подключений
- Пошаговое подтверждение каждого действия
- Валидация формата IP адреса
- Автоматическое определение подсети на основе текущего IP
- Предложение шлюза по умолчанию (первый адрес подсети)

### 🔧 Технические возможности

#### 1. Выбор сетевого подключения
```
Обнаружены сетевые подключения:
  1) Wired connection 1 (тип: ethernet, устройство: eth0)
  2) WiFi connection (тип: wireless, устройство: wlan0)
```

#### 2. Просмотр текущих настроек
```
Текущие настройки подключения:
  IP адрес:     192.168.1.50/24
  Шлюз:         192.168.1.1

  Метод:        auto
```

#### 3. Резервное копирование
```bash

$HOME/network_backups/Wired\ connection\ 1.20260329_143022.backup
```

#### 4. Валидация IP адреса
- Проверка формата (4 октета 0-255)
- Предупреждение при неверном вводе

#### 5. Выбор маски подсети
| Вариант | Маска | Количество хостов |
|---------|-------|-------------------|
| 1 | 255.255.255.0 (/24) | до 254 |
| 2 | 255.255.0.0 (/16) | до 65534 |
| 3 | 255.0.0.0 (/8) | до 16777214 |
| 4 | Своя маска | настраивается |

#### 6. Выбор DNS серверов
| Вариант | DNS серверы |
|---------|-------------|
| 1 | Google DNS (8.8.8.8, 8.8.4.4) |
| 2 | Cloudflare DNS (1.1.1.1, 1.0.0.1) |
| 3 | Яндекс DNS (77.88.8.8, 77.88.8.1) |
| 4 | Свои DNS серверы |

#### 7. Дополнительные IP адреса
```bash
# Добавление второго IP на интерфейс
nmcli connection modify "connection" \
    +ipv4.addresses "192.168.1.101/24"
```

#### 8. Проверка подключения
```bash
# Проверка связи со шлюзом
ping -c 2 -W 1 $GATEWAY

# Проверка работы DNS
nslookup google.com

# Проверка нового IP
hostname -I
```

#### 9. Скрипт отката к DHCP
```bash
#!/bin/bash
# ~/rollback_to_dhcp.sh
nmcli connection modify "Wired connection 1" ipv4.method auto
nmcli connection down "Wired connection 1" && nmcli connection up "Wired connection 1"
echo "✓ Возврат к DHCP выполнен"
```

## ⚠️ Важные замечания

1. **Резервное копирование**: Скрипт предлагает создать резервную копию перед внесением изменений.
2. **Проверка настроек**: После применения скрипт автоматически проверяет связь со шлюзом и работу DNS.
3. **Откат к DHCP**: При необходимости выполните команду для возврата к автоматическому получению IP:
   ```bash
   nmcli connection modify "имя_подключения" ipv4.method auto
   nmcli connection down "имя_подключения" && nmcli connection up "имя_подключения"
   ```
4. **Несколько IP**: Для добавления дополнительных адресов используйте опцию в скрипте или команду:
   ```bash
   nmcli connection modify "connection" +ipv4.addresses "192.168.1.101/24"
   ```
