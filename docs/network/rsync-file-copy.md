## Описание проблемы

При попытке скопировать файлы на внешний жёсткий диск возникает ошибка о слишком длинном пути/имени файла. Решение — использовать rsync через SSH-соединение.

---

## Исходные данные

| Параметр | Исходный ПК | Целевой ПК |
|----------|-------------|------------|
| **IP-адрес** | `192.168.1.100` | `192.168.1.200` |
| **Пользователь** | `user` | `user` |
| **Источник** | `/home/user/Документы/` | — |
| **Назначение** | — | `/home/user/Документы/target_folder/` |

---

## Предварительные требования

### 1. Установите rsync (на обоих ПК)

```bash
sudo dnf install rsync
```

### 2. Проверьте SSH-сервер на целевом ПК

```bash
sudo systemctl enable sshd
sudo systemctl start sshd
```

### 3. Проверьте доступность целевого ПК

```bash
ping 192.168.1.200
```

---

## Пошаговая инструкция

### Шаг 0: Проверьте доступ по SSH

⚠️ **Важно:** На некоторых системах РЕД ОС обычные пользователи (включая `root`) могут не иметь доступа по SSH из соображений безопасности. 

Если при подключении от имени обычного пользователя вы получаете `Permission denied`, используйте учетную запись администратора (например, `admin`), которая имеет SSH-доступ.

Проверьте доступность SSH:

```bash
# Проверка для обычного пользователя (может не сработать)
ssh user@192.168.1.200

# Проверка для администратора
ssh admin@192.168.1.200
```

---

### Шаг 1: Определите точное имя исходной папки

На **исходном ПК** выполните:

```bash
ls -la /home/user/
```

⚠️ **Важно:** Обратите внимание на регистр! Папка может называться `Документы` (с заглавной буквы), а не `документы`.

---

### Шаг 2: Создайте целевую папку на целевом ПК

На **целевом ПК** (192.168.1.200) выполните:

```bash
mkdir -p /home/user/Документы/target_folder
```

---

### Шаг 3: Скопируйте файлы через промежуточную папку

#### Почему не напрямую?

При попытке копирования напрямую от имени пользователя `admin` в папку `/home/user/` возникает ошибка **Permission denied (13)**, так как `admin` не имеет прав на запись в домашнюю директорию другого пользователя.

#### Решение: Копирование через промежуточную папку

**На исходном ПК** (192.168.1.100) выполните:

```bash
rsync -avz --progress /home/user/Документы/ admin@192.168.1.200:/home/admin/Документы_backup/
```

Система запросит пароль пользователя `admin` на целевом ПК.

---

### Шаг 4: Переместите файлы в целевую папку

На **целевом ПК** (192.168.1.200) выполните:

```bash
# Переместить папку
sudo mv /home/admin/Документы_backup /home/user/Документы/target_folder

# Исправить владельца
sudo chown -R user:user /home/user/Документы/target_folder
```

---

## Разбор команды rsync

```bash
rsync -avz --progress /путь/к/источнику/ пользователь@IP:/путь/к/назначению/
```

| Параметр | Описание |
|----------|----------|
| `-a` | Архивный режим (сохраняет права, временные метки, символические ссылки) |
| `-v` | Подробный вывод (verbose) |
| `-z` | Сжатие данных при передаче |
| `--progress` | Отображение прогресса копирования |
| `/источник/` | **Слэш в конце** = копировать содержимое папки |
| `/источник` | **Без слэша** = копировать саму папку |

---

## Права на файлы

### Что сохраняется с флагом `-a`:

- ✅ Права доступа (чтение, запись, выполнение)
- ✅ Временные метки (дата создания/изменения)
- ✅ Символические ссылки
- ✅ Структура каталогов

### Что изменяется:

- ❌ **Владелец файлов** станет тем пользователем, от имени которого выполняется копирование
- ❌ **Группа** также изменится

### Исправление владельца:

```bash
sudo chown -R user:user /home/user/Документы/target_folder
```

## Альтернативные варианты

### Вариант A: Прямое копирование от имени целевого пользователя

```bash
rsync -avz --progress /home/user/Документы/ user@192.168.1.200:/home/user/Документы/target_folder/
```

⚠️ Требуется пароль пользователя `user`.

---

### Вариант B: Предварительная настройка прав

На **целевом ПК**:

```bash
sudo mkdir -p /home/user/Документы/target_folder
sudo chown admin:admin /home/user/Документы/target_folder
```

Затем на **исходном ПК**:

```bash
rsync -avz --progress /home/user/Документы/ admin@192.168.1.200:/home/user/Документы/target_folder/
```

---

### Вариант C: Через промежуточную папку (Рекомендуемый)

Описан в основной инструкции выше. ✅

## Дополнительные возможности

### Продолжение прерванного копирования

```bash
rsync -avz --progress --partial /home/user/Документы/ admin@192.168.1.200:/home/admin/Документы_backup/
```

### Ускоренное копирование

```bash
rsync -avz --progress --compress-level=9 -e "ssh -c aes128-gcm@openssh.com" /home/user/Документы/ admin@192.168.1.200:/home/admin/Документы_backup/
```

### Копирование с сохранением владельцев (требует sudo)

```bash
sudo rsync -avz --progress --owner --group /home/user/Документы/ admin@192.168.1.200:/home/admin/Документы_backup/
```

## Частые ошибки

### 1. Permission denied (при подключении)

**Причина:** Неправильный пароль пользователя.

**Решение:** Убедитесь, что вводите пароль правильного пользователя.

```bash
# Проверка подключения
ssh admin@192.168.1.200
```

---

### 2. No such file or directory

**Причина:** Неправильный путь или регистр имени папки.

**Решение:** Проверьте точное имя папки:

```bash
ls -la /home/user/
```

---

### 3. Permission denied (13) (при записи)

**Причина:** Нет прав на запись в целевую папку.

**Решение:** Используйте промежуточную папку (Вариант C) или измените владельца целевой папки.

---

### 4. The authenticity of host can't be established

**Причина:** Первое подключение к этому ПК по SSH.

**Решение:** Введите `yes` для добавления ключа:

```
Are you sure you want to continue connecting (yes/no/[fingerprint])? yes
```

## Проверка результата

На **целевом ПК** выполните:

```bash
# Проверка структуры папок
ls -la /home/user/Документы/target_folder/

# Проверка владельца
ls -l /home/user/Документы/target_folder/

# Подсчёт файлов
find /home/user/Документы/target_folder/ -type f | wc -l
```

## Примечания

- РЕД ОС основана на Fedora/RHEL, используются команды `dnf` и `systemd`
- Имена файлов и папок на кириллице обрабатываются корректно
- При копировании больших объёмов рекомендуется использовать `screen` или `tmux` для защиты от обрыва соединения

```bash
# Установка screen
sudo dnf install screen

# Запуск сессии
screen

# Запуск rsync внутри screen
rsync -avz --progress /home/user/Документы/ admin@192.168.1.200:/home/admin/Документы_backup/

# Отключение от screen (Ctrl+A, D)
# Повторное подключение: screen -r
```
