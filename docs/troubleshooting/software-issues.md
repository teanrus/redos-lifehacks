## 🛠️ Лайфхаки по решению ошибок установки

### 1. Ошибка зависимости пакета

При установке пакета не хватает зависимостей:

```bash
# Пример ошибки:
# Error: Package: package-name-1.0.x86_64
#Requires: libfoo >= 2.0

# Автоматическое разрешение зависимостей
sudo dnf install --allowerasing package-name

# Принудительная установка с зависимостями
sudo dnf install -y package-name libfoo libbar

# Поиск недостающей зависимости
dnf provides libfoo.so.2

# Установка из конкретного репозитория
sudo dnf install --enablerepo=updates package-name
```

**Если зависимость отсутствует в репозиториях:**

```bash
# Поиск пакета с зависимостью
dnf search libfoo

# Установка из RPM-файла
sudo dnf install -y ./libfoo-2.0.rpm

# Игнорирование зависимости (не рекомендуется!)
sudo rpm -ivh --nodeps package-name.rpm
```

| Команда | Описание |
|---------|----------|
| `--allowerasing` | Удалить конфликтующие пакеты |
| `dnf provides` | Поиск пакета по файлу |
| `--enablerepo` | Включить репозиторий |
| `--nodeps` | Игнорировать зависимости (опасно!) |

> **Зачем:** Решение 90% ошибок при установке ПО.

### 2. Ошибка конфликта пакетов

Два пакета конфликтуют друг с другом:

```bash
# Пример ошибки:
# Error: Transaction test error:
#   file /usr/bin/example from install of package-new-1.0
#   conflicts with file from package old-2.0

# Удаление конфликтующего пакета
sudo dnf remove old-package

# Установка нового пакета
sudo dnf install new-package

# Или замена (обновление)
sudo dnf update old-package new-package

# Принудительная замена
sudo dnf install --allowerasing new-package
```

**Поиск конфликтующих файлов:**

```bash
# Какой пакет владеет файлом
rpm -qf /usr/bin/example

# Список файлов пакета
rpm -ql package-name

# Проверка установленных пакетов
rpm -qa | grep package
```

> **Зачем:** Решение конфликтов между версиями пакетов.

### 3. Ошибка места на диске

Недостаточно места для установки:

```bash
# Проверка места на диске
df -h
df -h /var

# Очистка кэша DNF
sudo dnf clean all
sudo rm -rf /var/cache/dnf/*

# Удаление старых ядер
sudo dnf remove $(dnf repoquery --installonly --latest-limit=-1 -q)

# Удаление ненужных пакетов
sudo dnf autoremove

# Поиск больших файлов
sudo du -ah /var | sort -rh | head -10
```

**Очистка логов:**

```bash
# Очистка старых логов
sudo journalctl --vacuum-time=2d

# Проверка размера логов
du -sh /var/log/
```

| Действие | Эффект |
|----------|--------|
| `dnf clean all` | Освобождение 100-500 МБ |
| Удаление старых ядер | 50-200 МБ за ядро |
| `dnf autoremove` | Зависит от системы |
| Очистка логов | До нескольких ГБ |

> **Зачем:** Освобождение места для установки пакетов.

### 4. Ошибка повреждённого пакета

Пакет загрузился с ошибками:

```bash
# Очистка кэша и повторная загрузка
sudo dnf clean all
sudo rm -rf /var/cache/dnf/*
sudo dnf makecache
sudo dnf install package-name

# Проверка целостности RPM
rpm -K package-name.rpm

# Повторная загрузка с другого зеркала
sudo dnf install --downloaddir=/tmp/package package-name

# Установка с проверкой контрольной суммы
sudo dnf install --check package-name
```

**Если проблема с репозиторием:**

```bash
# Очистка метаданных
sudo dnf clean metadata

# Обновление кэша
sudo dnf makecache

# Проверка репозитория
dnf repolist -v
```

> **Зачем:** Решение проблем с неполной загрузкой пакетов.

### 5. Ошибка репозитория (404, не найден)

Репозиторий недоступен или перемещён:

```bash
# Пример ошибки:
# Error: Failed to download metadata for repo 'example':
#Status code: 404

# Отключение проблемного репозитория
sudo dnf config-manager --set-disabled example

# Обновление списка репозиториев
sudo dnf clean all
sudo dnf makecache

# Проверка доступности репозитория
curl -I https://example.com/repo/

# Редактирование конфига репозитория
sudo nano /etc/yum.repos.d/example.repo
```

**Временное игнорирование репозитория:**

```bash
# Установка с игнорированием репозитория
sudo dnf install --disablerepo=example package-name

# Постоянное отключение
sudo dnf config-manager --set-disabled example
```

> **Зачем:** Продолжение работы при недоступности репозитория.

### 6. Ошибка подписи GPG

Неверная или отсутствующая подпись пакета:

```bash
# Пример ошибки:
# Error: GPG check FAILED

# Импорт ключа репозитория
sudo rpm --import /etc/pki/rpm-gpg/RPM-GPG-KEY-redos

# Импорт ключа из интернета
sudo rpm --import https://example.com/RPM-GPG-KEY

# Отключение проверки GPG (не рекомендуется!)
sudo dnf install --nogpgcheck package-name

# Проверка ключей
rpm -qa gpg-pubkey*
```

**Постоянное отключение проверки GPG (только для доверенных репозиториев):**

```bash
# Редактирование репозитория
sudo nano /etc/yum.repos.d/example.repo

# Добавить строку
gpgcheck=0
```

> **⚠️ Внимание:** Отключение проверки GPG снижает безопасность!

### 7. Ошибка «Пакет уже установлен»

Пакет уже есть в системе:

```bash
# Пример ошибки:
# Package package-name-1.0.x86_64 is already installed.

# Проверка установленной версии
rpm -q package-name

# Обновление до последней версии
sudo dnf update package-name

# Переустановка
sudo dnf reinstall package-name

# Установка конкретной версии
sudo dnf install package-name-2.0
```

**Если нужна другая версия:**

```bash
# Поиск доступных версий
dnf list --showduplicates package-name

# Установка конкретной версии
sudo dnf install package-name-1.5-1.el8

# Понижение версии (даунгрейд)
sudo dnf downgrade package-name
```

> **Зачем:** Управление версиями установленных пакетов.

### 8. Ошибка при обновлении системы

Проблемы при `dnf update`:

```bash
# Обновление с исключением проблемных пакетов
sudo dnf update --exclude=problematic-package

# Обновление без учёта зависимостей
sudo dnf distro-sync

# Откат транзакции
sudo dnf history
sudo dnf history undo last

# Построчное обновление (по одному пакету)
sudo dnf update package1
sudo dnf update package2
```

**Если обновление зависает:**

```bash
# Принудительная остановка
sudo killall -9 dnf
sudo killall -9 rpm

# Удаление lock-файлов
sudo rm -f /var/lib/rpm/__db*

# Пересоздание базы RPM
sudo rpm --rebuilddb

# Повторное обновление
sudo dnf update
```

> **Зачем:** Решение проблем с зависанием обновления.

### 9. Ошибка «Невозможно получить lock»

Блокировка базы данных пакетов:

```bash
# Пример ошибки:
# Error: Could not get lock /var/lib/dnf/lock
#Unable to open database

# Проверка процессов DNF/RPM
ps aux | grep -E 'dnf|rpm'

# Завершение процессов
sudo killall -9 dnf
sudo killall -9 rpm

# Удаление lock-файлов
sudo rm -f /var/lib/dnf/lock
sudo rm -f /var/lib/rpm/__db*

# Пересоздание базы RPM
sudo rpm --rebuilddb

# Очистка кэша
sudo dnf clean all
```

**Автоматический скрипт разблокировки:**

```bash
#!/bin/bash
sudo killall -9 dnf rpm 2>/dev/null
sudo rm -f /var/lib/dnf/lock /var/lib/rpm/__db*
sudo rpm --rebuilddb
sudo dnf clean all
echo "Блокировка снята!"
```

> **Зачем:** Решение проблемы с заблокированной базой пакетов.

### 10. Ошибка 32-битных пакетов на 64-битной системе

Проблемы с архитектурой пакетов:

```bash
# Пример ошибки:
# Error: package-name.i686 does not match your architecture

# Проверка архитектуры системы
uname -m
rpm --eval '%{_arch}'

# Включение поддержки i686 (если нужно)
sudo dnf config-manager --set-enabled crb  # для RHEL 8+

# Установка с указанием архитектуры
sudo dnf install package-name.i686

# Поиск 64-битной версии
dnf search package-name | grep x86_64
```

**Настройка мультиархитектурности:**

```bash
# Добавление i686 архитектуры
sudo rpm --import /etc/pki/rpm-gpg/RPM-GPG-KEY-redos
sudo dnf config-manager --set-enabled powertools

# Установка 32-битных библиотек
sudo dnf install glibc.i686 libgcc.i686
```

| Архитектура | Описание |
|-------------|----------|
| `x86_64` | 64-битная (стандарт) |
| `i686` | 32-битная (устаревшая) |
| `aarch64` | ARM 64-битная |
| `noarch` | Не зависит от архитектуры |

> **Зачем:** Установка 32-битных приложений на 64-битную систему.

## ⚠️ Важные замечания

1. **GPG подписи**: Не отключайте проверку GPG без необходимости!
2. **--nodeps**: Игнорирование зависимостей может сломать систему.
3. **Lock-файлы**: Удаляйте только при уверенности, что DNF не работает.
4. **Очистка логов**: Сохраняйте важные логи перед очисткой.
5. **Репозитории**: Отключайте только проблемные, не все сразу.
6. **Бэкап**: Перед серьёзными изменениями сделайте резервную копию.
