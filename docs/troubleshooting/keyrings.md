# Сброс связки ключей GNOME Keyring в РЕД ОС

Инструкция по разблокировке и сбросу связки ключей в операционной системе РЕД ОС.

## 📋 Описание

GNOME Keyring — это система хранения паролей и ключей шифрования в Linux. При входе в систему она автоматически разблокируется паролем пользователя. Если пароль был изменён или возникли проблемы с разблокировкой, требуется сброс связки ключей.

---

## ⚠️ Важное предупреждение

**Перед сбросом ознакомьтесь с последствиями:**

| Что будет потеряно | Можно ли восстановить |
|-------------------|----------------------|
| Пароли браузеров (Chrome, Firefox, Yandex) | ❌ Нет (вводите заново) |
| Пароли Wi-Fi сетей | ❌ Нет (подключайтесь заново) |
| Пароли почтовых клиентов | ❌ Нет (вводите заново) |
| SSH-ключи (если хранятся в keyring) | ✅ Да (файлы в `~/.ssh/` не затрагиваются) |
| Сертификаты GPG | ✅ Да (файлы в `~/.gnupg/` не затрагиваются) |

---

## 🎯 Симптомы проблем со связкой ключей

| Симптом | Вероятная причина |
|---------|-------------------|
| Постоянные запросы пароля при входе | Пароль пользователя изменён |
| Сообщение «Не удалось разблокировать связку ключей» | Несоответствие паролей |
| Браузер не запоминает пароли | Повреждение keyring |
| Запрос пароля keyring при каждом запуске браузера | Автоматическая разблокировка не работает |

## 🚀 Способы сброса

### Способ 1: Полное удаление (рекомендуется при забытом пароле)

Этот метод полностью удаляет все связки ключей. Система создаст новые при следующем входе.

#### Шаг 1: Выйдите из системы

Завершите все сеансы пользователя:

```bash
# Завершить все процессы пользователя (из-под root)
sudo pkill -u username
```

Или выйдите через графический интерфейс.

#### Шаг 2: Удалите файлы связки ключей

```bash
# Перейдите в домашний каталог
cd ~

# Удалите директорию keyrings
rm -rf ~/.local/share/keyrings/
```

**Или удалите выборочно:**

```bash
# Удалить только основную связку (login.keyring)
rm ~/.local/share/keyrings/login.keyring

# Удалить связку по умолчанию
rm ~/.local/share/keyrings/default.keyring
```

#### Шаг 3: Перезапустите демон keyring (опционально)

```bash
# Завершить процесс демона
killall gnome-keyring-daemon

# Или принудительно
killall -9 gnome-keyring-daemon
```

#### Шаг 4: Войдите в систему заново

При следующем входе система:
- Автоматически создаст новую связку ключей
- Запросит новый пароль (или предложите оставить пустым)

---

### Способ 2: Через графический интерфейс (если помните старый пароль)

#### Шаг 1: Откройте приложение «Связка ключей»

```bash
# Через терминал
seahorse

# Или через меню приложений:
# Приложения → Стандартные → Связка ключей
```

#### Шаг 2: Измените пароль

1. Найдите связку **«Вход»** (или **«Login»**) в списке слева
2. Щёлкните правой кнопкой мыши → **«Изменить пароль»**
3. Введите **текущий пароль**
4. Введите **новый пароль** (или оставьте пустым для отключения)
5. Нажмите **«Продолжить»**

#### Шаг 3: Разблокируйте связку

Если связка заблокирована:
1. Щёлкните правой кнопкой → **«Разблокировать»**
2. Введите пароль
3. Установите галочку **«Разблокировать автоматически при входе»**

---

### Способ 3: Сброс через терминал (альтернативный)

#### Шаг 1: Остановите демон keyring

```bash
# Завершить все процессы gnome-keyring
pkill gnome-keyring
```

#### Шаг 2: Переименуйте существующие файлы (для резервного копирования)

```bash
mkdir -p ~/keyrings-backup
mv ~/.local/share/keyrings/*.keyring ~/keyrings-backup/
```

#### Шаг 3: Очистите кэш сессии

```bash
# Удалить файлы сессии (если существуют)
rm -rf /run/user/$(id -u)/keyring/
```

#### Шаг 4: Перезапустите систему

```bash
sudo reboot
```

## 🔧 Настройка автоматической разблокировки

Чтобы связка ключей разблокировалась автоматически при входе:

### Требование

**Пароль связки ключей должен совпадать с паролем пользователя.**

### Если пароли не совпадают

```bash
# 1. Измените пароль пользователя
sudo passwd username

# 2. Измените пароль связки ключей на тот же самый
# (через Seahorse или удалите и создайте заново)
rm ~/.local/share/keyrings/login.keyring
```

### Проверка автоматической разблокировки

1. Выйдите из системы
2. Войдите с новым паролем
3. Проверьте, что нет запросов пароля keyring

## ⚠️ Возможные проблемы и решения

### Проблема 1: Постоянные запросы пароля после входа

**Причина:** Пароль связки не совпадает с паролем пользователя.

**Решение:**

```bash
# Удалить связку и создать заново
rm -rf ~/.local/share/keyrings/
# Перезайти в систему
```

### Проблема 2: Браузер не запоминает пароли

**Причина:** Keyring не разблокирован или повреждён.

**Решение:**

```bash
# Проверить процесс keyring
ps aux | grep gnome-keyring

# Перезапустить
killall gnome-keyring-daemon
# Перезайти
```

### Проблема 3: Ошибка «Не удалось создать связку ключей»

**Причина:** Проблемы с правами доступа.

**Решение:**

```bash
# Исправить права
chown -R $USER:$USER ~/.local/share/keyrings/
chmod 700 ~/.local/share/keyrings/
```

### Проблема 4: Keyring запрашивает пароль при запуске браузера

**Причина:** Автоматическая разблокировка отключена.

**Решение:**
1. Откройте **Seahorse**
2. Правой кнопкой на **«Вход»** → **«Разблокировать»**
3. Введите пароль
4. Установите **«Разблокировать при входе»**

## 📁 Структура файлов GNOME Keyring

```
~/.local/share/keyrings/
├── login.keyring      # Основная связка (пароли браузеров, Wi-Fi)
├── default.keyring    # Связка по умолчанию
└── user.keystore      # Пользовательское хранилище
```

**Сопутствующие файлы:**

```
~/.config/autostart/
└── gnome-keyring-pkcs11.desktop  # Автозапуск

/run/user/<uid>/keyring/          # Сессионные данные
```

## 🔍 Диагностика

### Проверка состояния keyring

```bash
# Проверить процесс
ps aux | grep gnome-keyring

# Проверить файлы
ls -la ~/.local/share/keyrings/

# Проверить права
namei -l ~/.local/share/keyrings/
```

### Просмотр логов

```bash
# Логи сессии
journalctl --user | grep keyring

# Системные логи
sudo journalctl | grep gnome-keyring
```

### Тестирование

```bash
# Запустить демон вручную
gnome-keyring-daemon --start --components=secrets

# Проверить сокет
echo $GNOME_KEYRING_CONTROL
```

## 🛡️ Безопасность

### Рекомендации

| Действие | Рекомендация |
|----------|-------------|
| Пароль связки | Должен совпадать с паролем пользователя |
| Пустой пароль | ⚠️ Не рекомендуется (снижает безопасность) |
| Резервное копирование | Сохраняйте `login.keyring` перед сбросом |
| Доменная учётная запись | Может потребоваться настройка Kerberos |

### Если используется домен Active Directory

```bash
# Проверить интеграцию с доменом
realm list

# При проблемах может потребоваться:
sudo pam-config --add --pamkrb5
```

## 📝 Восстановление после сброса

### Что нужно сделать заново

1. **Браузеры:**
   - Ввести пароли от сайтов
   - Включить синхронизацию (если используется)

2. **Wi-Fi:**
   - Подключиться к сетям заново
   - Ввести пароли

3. **Почтовые клиенты:**
   - Thunderbird, Evolution — ввести пароли

4. **Мессенджеры:**
   - Некоторые могут запросить повторную авторизацию

### Ускорение восстановления

```bash
# Для браузеров на базе Chromium можно включить синхронизацию
# и пароли восстановятся из облака

# Firefox: войти в Firefox Account
```

# Версия пакета
rpm -q gnome-keyring

sudo dnf reinstall gnome-keyring

rm -rf ~/.config/gnome-keyring/
