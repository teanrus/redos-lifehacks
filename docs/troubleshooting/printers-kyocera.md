## Проблема с печатью из РЕД ОС
Проблема с печатью из РЕД ОС возникает из-за отсутствия официальных драйверов на сайте Kyocera (они недоступны для загрузки в РФ). Однако для Linux существует проверенное решение, которое работает и в РЕД ОС, так как она основана на CentOS/Red Hat.

Основной вариант — использовать универсальный драйвер GDI через систему CUPS. Для этого подойдут драйверы из репозиториев на GitHub, созданные сообществом специально для вашей модели.

### На март 2026 года решение актуально для следующих моделей:
- Kyocera FS-1020MFP
- Kyocera FS-1025MFP
- Kyocera FS-1040
- Kyocera FS-1060DN
- Kyocera FS-1120MFP
- Kyocera FS-1125MFP

## ✅ Решение

### 🔧 Способ 1: Установка драйверов через автоматический скрипт (рекомендуется)
Этот метод наиболее простой и быстрый. Вам нужно открыть терминал (консоль) и выполнить команду для загрузки и запуска скрипта установки.
#### Скопируйте и вставьте в терминал следующую команду:
```bash
sudo wget -qO- https://raw.githubusercontent.com/ergiosko/kyocera-ppd/refs/heads/main/install | sudo bash
```
#### Что произойдет:
- Скрипт автоматически загрузит все необходимые PPD-файлы и фильтры для вашего принтера .
- Скопирует их в нужные системные папки (/usr/share/cups/model/Kyocera и /usr/lib/cups/filter) .
#### После завершения установки обязательно перезапустите службу печати CUPS:
```bash
sudo systemctl restart cups.service
```
### 🖥️ Способ 2: Ручная установка (если скрипт не сработал)
Если автоматический скрипт не подходит или не работает, вы можете установить драйвер вручную, используя другой скрипт-установщик .
#### Скачайте установщик с GitHub:
```bash
wget https://raw.githubusercontent.com/mnorin/kyocera-ppd-installer/master/kyocera_ppds_installer.sh
```
#### Сделайте файл исполняемым и запустите его с правами администратора:
```bash
chmod +x kyocera_ppds_installer.sh
sudo ./kyocera_ppds_installer.sh
```
#### После установки перезапустите CUPS:
```bash
sudo systemctl restart cups.service
```
### 🖨️ Настройка принтера после установки драйвера
После установки драйвера через терминал, принтер необходимо добавить в систему через веб-интерфейс CUPS:
- Откройте браузер и перейдите по адресу: http://localhost:631 .
- Перейдите в раздел Administration (Администрирование) и нажмите Add Printer (Добавить принтер).
- Введите имя и пароль пользователя системы (root или пользователь с правами sudo).
- В списке найдите ваш принтер Kyocera FS-1125MFP (он должен определиться по USB или сети).
- На шаге выбора драйвера найдите в списке производителя Kyocera, а в моделях — модель вашего принтера (в моем случае Kyocera FS-1125MFPGDI).
- Завершите настройку и отправьте тестовую страницу.

### 💡 Что делать, если не печатает?
- Проверьте порт: Убедитесь, что в настройках принтера в CUPS выбран правильный порт (например, usb://... или socket://... для сети).
- Права доступа: Убедитесь, что у пользователя РЕД ОС есть права на печать (обычно это настраивается в свойствах принтера в разделе "Политики").
- Перезагрузка: После всех манипуляций перезагрузите компьютер и принтер.

## 📡 Настройка сетевого принтера

Если принтер подключён к одному компьютеру по USB, другие компьютеры в локальной сети могут печатать на него через сетевой доступ CUPS.

### 🔧 Быстрая настройка (автоматический скрипт)

#### На сервере печати (ПК с подключённым принтером):

```bash
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/setup-network-printer.sh | sudo bash
```

Скрипт автоматически:
- Настроит CUPS для сетевого доступа
- Откроет порты в firewall (631, 9100)
- Включит общий доступ к принтеру
- Создаст пользователя для печати
- Создаст скрипт проверки статуса

#### На клиенте (другой ПК в сети):

1. Узнайте IP-адрес сервера печати:
   ```bash
   # На сервере выполнения
   ip addr show
   ```

2. Добавьте сетевой принтер через веб-интерфейс CUPS:
   - Откройте браузер: `http://localhost:631`
   - Administration → Add Printer
   - Выберите **Network Printer** или **IPP**
   - Введите адрес: `ipp://<IP-сервера>:631/printers/<имя_принтера>`
   - Выберите драйвер Kyocera FS-1125MFP

3. Или через терминал:
   ```bash
   lpadmin -p Network-Kyocera -v ipp://<IP-сервера>:631/printers/<имя_принтера> -E
   ```

### 🖥️ Ручная настройка

#### Шаг 1: Настройка сервера печати

На компьютере с подключённым принтером:

```bash
# 1. Отредактировать конфигурацию CUPS
sudo nano /etc/cups/cupsd.conf
```

Найдите и измените:
```conf
# Слушать все интерфейсы
Port 631
Listen /var/run/cups/cups.sock

# Разрешить доступ из локальной сети
<Location />
  Order allow,deny
  Allow @LOCAL
</Location>

<Location /admin>
  Order allow,deny
  Allow @LOCAL
  AuthType Default
  Require user @SYSTEM
</Location>

<Location /admin/conf>
  AuthType Default
  Require user @SYSTEM
  Order allow,deny
  Allow @LOCAL
</Location>
```

```bash
# 2. Включить общий доступ к принтерам
sudo cupsctl --share-printers

# 3. Открыть порты в firewall
sudo firewall-cmd --permanent --add-service=ipp
sudo firewall-cmd --permanent --add-service=ipps
sudo firewall-cmd --permanent --add-port=9100/tcp
sudo firewall-cmd --reload

# 4. Перезапустить CUPS
sudo systemctl restart cups
```

#### Шаг 2: Узнать IP-адрес сервера

```bash
ip addr show
# Запомните IP (например, 192.168.1.100)
```

#### Шаг 3: Подключение клиента

**Вариант А: Через веб-интерфейс CUPS**

1. Откройте браузер: `http://localhost:631`
2. **Administration** → **Add Printer**
3. Выберите **Network Printer** или **AppSocket/HP Jetdirect**
4. Введите адрес:
   - `socket://<IP-сервера>:9100` или
   - `ipp://<IP-сервера>:631/printers/<имя_принтера>`
5. Выберите драйвер Kyocera FS-1125MFP
6. Завершите настройку

**Вариант Б: Через терминал**

```bash
# Добавить принтер
sudo lpadmin -p Network-Kyocera -v socket://<IP-сервера>:9100 -E

# Установить по умолчанию (опционально)
sudo lpoptions -d Network-Kyocera

# Проверить
lpstat -p
```

**Вариант В: Через графический интерфейс**

```bash
system-config-printer
```

1. Нажмите **Добавить принтер**
2. Выберите **Сетевой принтер**
3. Укажите протокол и адрес:
   - **AppSocket/HP Jetdirect**: `socket://<IP-сервера>:9100`
   - **IPP**: `ipp://<IP-сервера>:631/printers/<имя_принтера>`
4. Выберите драйвер Kyocera FS-1125MFP

### 🔍 Проверка

```bash
# Проверить доступность принтера
lpstat -p

# Отправить тестовую страницу
lp -d <имя_принтера> /usr/share/cups/data/testprint

# Проверить очередь
lpstat -o

# Статус службы CUPS
systemctl is-active cups
```

### ⚠️ Возможные проблемы

| Проблема | Решение |
|----------|---------|
| Принтер не виден в сети | Проверьте firewall: `sudo firewall-cmd --list-all` |
| Ошибка подключения | Убедитесь, что CUPS слушает сеть: `sudo netstat -tlnp \| grep 631` |
| Нет драйвера на клиенте | Установите драйвер по инструкции выше |
| Запрос пароля | Создайте пользователя: `sudo lpadmin -p <принтер> -u allow:<пользователь>` |
| Принтер офлайн | Проверьте: `lpstat -p <принтер>` |

## 📝 Полезные команды

| Команда | Описание |
|---------|----------|
| `lpstat -p` | Список принтеров |
| `lpstat -o` | Очереди печати |
| `lpstat -d` | Принтер по умолчанию |
| `lp -d <принтер> <файл>` | Печать файла |
| `cancel <задание>` | Отмена задания |
| `cupsctl --share-printers` | Включить общий доступ |
| `systemctl restart cups` | Перезапуск CUPS |
| `system-config-printer` | Графическая настройка |
