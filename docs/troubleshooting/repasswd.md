# Восстановление пароля root в РЕД ОС 7.3

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](/LICENSE)
[![Platform](https://img.shields.io/badge/platform-RED%20OS%207.3-red.svg)](https://redos.red-soft.ru/)
[![Stars](https://img.shields.io/github/stars/teanrus/redos-lifehacks.svg)](https://github.com/teanrus/redos-lifehacks/stargazers)

> **Важно:** Для выполнения этой инструкции требуется физический доступ к консоли сервера или доступ к remote console виртуальной машины.

---

## Основной способ: через GRUB с параметром `init=/bin/bash`

Этот метод проверен на РЕД ОС 7.3 и является наиболее надёжным.

### Шаг 1. Редактирование загрузочного меню GRUB

1. Перезагрузите сервер
2. Как только появится меню загрузчика GRUB — быстро нажмите клавиши **↑** или **↓**, чтобы остановить отсчёт времени
3. Выберите нужный ядро и нажмите клавишу **e** для редактирования параметров
4. Найдите строку, начинающуюся с **`linux`** (для РЕД ОС 7.x)
5. В конец этой строки добавьте параметр:

   ```bash
   init=/bin/bash
   ```

   **Пример строки после редактирования:**

   ```bash
   linux /vmlinuz-... rhgb quiet init=/bin/bash
   ```

6. Нажмите **Ctrl+X** или **F10** для загрузки с изменёнными параметрами

---

### Шаг 2. Смена пароля root

После загрузки вы увидите командную строку bash:

```bash
bash-4.2#
```

Выполните команды по порядку:

```bash
# 1. Перемонтировать корневую файловую систему в режим чтения-записи
mount -o remount,rw /

# 2. Сменить пароль root
passwd root
```

Введите новый пароль дважды при появлении запроса.  
> ⚠️ Символы пароля не отображаются при вводе — это нормально.

```bash
# 3. Обновить контекст безопасности SELinux (ОБЯЗАТЕЛЬНО!)
touch /.autorelabel

# 4. Синхронизировать данные на диске
sync

# 5. Запустить систему
exec /sbin/init
```

***Или выполните принудительную перезагрузку:***

```bash
reboot -f
```

---

### Шаг 3. Первая загрузка после сброса

При первой загрузке после смены пароля:

- SELinux выполнит перемаркировку файловой системы
- Это может занять **от 2 до 10 минут** в зависимости от размера диска
- На экране может отображаться прогресс-бар или сообщение о перемаркировке
- **Не прерывайте процесс** — дождитесь полной загрузки системы

После завершения перемаркировки система загрузится в обычном режиме, и вы сможете войти под root с новым паролем.

## Краткая шпаргалка

```bash
# В GRUB: добавить init=/bin/bash в строку linux

# В bash после загрузки:
mount -o remount,rw /
passwd root
touch /.autorelabel
sync
exec /sbin/init
```

## Альтернативные способы (для справки)

### Способ A: Через параметр `rd.break`

1. В GRUB добавьте параметр `rd.break` вместо `init=/bin/bash`
2. После загрузки в emergency shell выполните:

```bash
mount -o remount,rw /sysroot
chroot /sysroot
passwd root
touch /.autorelabel
exit
exit
```

---

### Способ B: Через LiveCD / Rescue-режим

Используйте, если методы через GRUB не работают.

1. Загрузитесь с установочного ISO/USB РЕД ОС
2. Выберите режим восстановления (Rescue mode)
3. Выполните:

```bash
# Определить корневой раздел
lsblk

# Смонтировать систему (замените устройство на своё)
mkdir /mnt/sysimage
mount /dev/mapper/VolGroup00-LogVol00 /mnt/sysimage
mount -t proc /proc /mnt/sysimage/proc
mount --rbind /dev /mnt/sysimage/dev
mount --rbind /sys /mnt/sysimage/sys

# Войти в систему и сменить пароль
chroot /mnt/sysimage
passwd root
touch /.autorelabel

# Выйти и перезагрузиться
exit
umount /mnt/sysimage/proc
umount /mnt/sysimage/dev
umount /mnt/sysimage/sys
umount /mnt/sysimage
reboot
```

## Возможные проблемы

| Проблема | Решение |
| -------- | ------- |
| Система не загружается после смены пароля | Не создан `/.autorelabel` — повторите процедуру |
| Команда `passwd` зависает | Установите локаль: `export LANG=en_US.UTF-8` |
| Диск зашифрован LUKS | Методы через GRUB не работают — нужен ключ дешифрования |
| GRUB защищён паролем | Требуется пароль загрузчика или редактирование с LiveCD |
| PAM блокирует вход | Выполните: `faillock --user root --reset` |

---

## Примечания

- Метод проверен на **РЕД ОС 7.3**
- Файл `/.autorelabel` будет автоматически удалён после завершения перемаркировки SELinux
- После восстановления пароля рекомендуется проверить логи: `journalctl -xb`
