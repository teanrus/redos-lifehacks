# Миграция пользователя в РЕД ОС

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](/LICENSE)
[![Platform](https://img.shields.io/badge/platform-RED%20OS%207.x-orange.svg)](https://redos.red-soft.ru/)

[![Stars](https://img.shields.io/github/stars/teanrus/redos-lifehacks.svg)](https://github.com/teanrus/redos-lifehacks/stargazers)

>Скрипт `user-migration.sh` предназначен для миграции учётной записи пользователя в операционной системе РЕД ОС.
>Автоматизирует создание нового пользователя, перенос файлов и настройку прав доступа.

---

## 🎯 Возможности

### ✅ Автоматизация
- Создание нового пользователя с комментариями (ФИО, должность)
- Автоматическая установка пароля
- Добавление в группу `wheel` для доступа к `sudo`

### ✅ Перенос данных
- Все файлы из домашнего каталога
- Скрытые конфигурационные файлы
- Данные браузеров (закладки, пароли, история, cookies)

### ✅ Поддерживаемые браузеры
| Браузер | Данные |
|---------|--------|
| Google Chrome | ✅ |
| Chromium | ✅ |
| Mozilla Firefox | ✅ |
| Яндекс.Браузер | ✅ |
| Opera | ✅ |

### ✅ Безопасность
- Перенос ключей GNOME Keyring (для расшифровки паролей)
- Корректная установка прав доступа
- Сохранение прав на SSH-ключи

### ✅ Интерактивность
- Выбор старого пользователя из списка
- Ввод данных нового пользователя
- Подтверждение каждого этапа
- Опция удаления старого пользователя


```bash

```

Скрипт проверит права доступа и отобразит приветственное сообщение.

### Шаг 2: Выбор старого пользователя

Скрипт автоматически получит список всех обычных пользователей системы (UID ≥ 1000) и предложит нумерованный список:

```
👤 ВЫБОР СТАРОГО ПОЛЬЗОВАТЕЛЯ
============================================

Список пользователей системы:

      Домашний каталог: /home/SidorovII

      Домашний каталог: /home/PetrovPA
   3) SemkinaSY (Сёмкина Светлана Юрьевна)
      Домашний каталог: /home/SemkinaSY

Введите номер пользователя для переноса данных (1-3): 3
```

### Шаг 3: Ввод данных нового пользователя

Скрипт запросит данные для создания нового пользователя:

```
============================================
👤 ДАННЫЕ НОВОГО ПОЛЬЗОВАТЕЛЯ
============================================

Введите логин нового пользователя (латиницей): ivanovii

Введите ФИО нового пользователя (например: Иванов Иван Иванович): Иванов Иван Иванович

Введите должность или комментарий (например: Ведущий специалист): Ведущий специалист
```

**Требования к логину:**
- Только латинские буквы
- Должен начинаться с буквы
- Допускаются цифры, `-` и `_`

Скрипт отобразит сводку параметров:

```
-------------------------------------------

💼 Должность: Ведущий специалист

Продолжить миграцию? (y/n): y
```

### Шаг 5: Добавление в группу wheel

Скрипт предложит добавить нового пользователя в группу администраторов:

```
📋 Шаг 3/7: Добавление в группы...

Группа 'wheel' предоставляет права администратора (доступ к sudo)

Добавить пользователя 'ivanovii' в группу 'wheel'? (y/n): y

✅ Добавлен в группу 'wheel' (sudo)
```

**Что такое wheel:**
- Группа для пользователей с правами администратора
- Позволяет выполнять команды с правами root через `sudo`
- Рекомендуется для ведущих специалистов и администраторов

**Если выбрали `n`:**
- Пользователь не будет иметь прав администратора
- Для добавления позже: `sudo usermod -aG wheel ivanovii`

## 📊 Что делает скрипт

### Этапы работы

| Шаг | Действие | Описание |
|-----|----------|----------|
| 1 | Создание пользователя | `useradd -m -c "ФИО, должность" login` |
| 2 | Установка пароля | `passwd login` |
| 3 | Добавление в группы | Запрос: добавить ли в группу `wheel` |
| 4 | Перенос файлов | `rsync -avh /home/old/ /home/new/` |
| 5 | Перенос браузеров | Копирование профилей 5 браузеров + keyrings |
| 6 | Исправление прав | `chown -R new:new /home/new/` |
| 7 | Проверка | Вывод информации о пользователе и размера данных |

### Переносимые данные

**Домашний каталог:**
- Все файлы и папки
- Скрытые файлы (`.bashrc`, `.profile`, `.gitconfig` и др.)

**Браузеры:**
- `~/.config/google-chrome/` — Google Chrome
- `~/.config/chromium/` — Chromium
- `~/.mozilla/firefox/` — Mozilla Firefox
- `~/.config/Yandex/YandexBrowser/` — Яндекс.Браузер
- `~/.config/opera/` — Opera

**Ключи шифрования:**
- `~/.local/share/keyrings/` — GNOME Keyring (пароли браузеров)

**SSH-ключи:**
- Сохраняются права `700` на `.ssh/` и `600` на файлы внутри

## ⚠️ Важные замечания

### Перед запуском

1. **Убедитесь, что старый пользователь вышел из системы**
   ```bash
   who  # Проверка активных сессий
   ```

2. **Закройте все браузеры** (важно для корректного копирования данных)

3. **Проверьте свободное место**
   ```bash
   df -h /home
   ```

### Безопасность паролей браузеров

**Google Chrome / Chromium:**
- Пароли зашифрованы с использованием GNOME Keyring
- Скрипт автоматически переносит ключи из `~/.local/share/keyrings/`
- При первом входе может потребоваться ввести старый пароль ключа

**Mozilla Firefox:**
- Пароли хранятся в `logins.json` и `key4.db`
- Переносятся автоматически вместе с профилем

### Возможные проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| Ошибка прав доступа | Запуск не от root | Используйте `sudo ./user-migration.sh` |
| Файлы заблокированы | Пользователь в системе | Завершите сессию: `pkill -u username` |
| Недостаточно места | Заполнен `/home` | Освободите место: `df -h /home` |
| Браузер не открывает профиль | Неверные права | `chown -R new:new ~/.mozilla/` |

## 🔧 Дополнительные команды

### После миграции

```bash
# Проверить информацию о пользователе
id ivanovii

# Проверить группы
groups ivanovii

# Проверить домашний каталог
ls -la /home/ivanovii/

# Проверить размер данных
du -sh /home/ivanovii/
```

### Если старший пользователь не был удалён

```bash
# Удалить пользователя и его файлы
sudo userdel -r SemkinaSY

# Если пользователь активен, завершить процессы сначала
sudo pkill -9 -u SemkinaSY
sudo userdel -r SemkinaSY
```

### Восстановление после ошибки

```bash
# Удалить нового пользователя и начать заново
sudo userdel -r ivanovii

# Запустить скрипт повторно
sudo ./user-migration.sh
```
