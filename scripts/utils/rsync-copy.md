# Интерактивный скрипт копирования файлов

## Описание

Интерактивный Bash-скрипт для удобного копирования файлов и папок с локальной машины на удалённый компьютер по SSH с использованием rsync. Скрипт пошагово запрашивает все необходимые параметры и выполняет проверки на каждом этапе.

## Возможности

- ✅ Проверка существования исходной папки (с автоподбором при ошибке регистра)
- ✅ Проверка доступности целевого компьютера (ping)
- ✅ Проверка SSH-доступа к целевому компьютеру
- ✅ Проверка/создание целевой папки на удалённом хосте
- ✅ Гибкая настройка параметров копирования (архивный режим, сжатие, прогресс, ускорение)
- ✅ Подсчёт времени копирования
- ✅ Информативный отчёт о результатах

## Требования

- РЕД ОС 7.3 / 8.x
- Установленные утилиты: `rsync`, `sshpass` (устанавливаются автоматически при отсутствии)
- Активный SSH-сервер на целевом компьютере
- Сетевая доступность целевого компьютера

## Использование

### 🚀 Быстрый запуск

```bash
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/rsync-copy.sh | sudo bash
```

### Вариант: Скачивание и запуск

```bash
# Скачайте скрипт
wget https://github.com/teanrus/redos-lifehacks/releases/latest/download/rsync-copy.sh

# Сделайте исполняемым
chmod +x rsync-copy.sh

# Запустите от root
sudo ./rsync-copy.sh
```

> ⚠️ **Важно:** Скрипт необходимо запускать на **исходной машине** (откуда копируются файлы).

## Пошаговый процесс

### Шаг 1: Исходная папка

Скрипт запросит путь к папке, откуда копировать файлы.

```bash
Введите путь к исходной папке (например, /home/user/Документы):
```

- Проверяется существование папки
- Если папка не найдена, автоматически ищется похожая (с учётом регистра)
- Предлагается использовать найденный вариант

### Шаг 2: IP целевого компьютера

```bash
Введите IP-адрес целевого компьютера (например, 192.168.1.200):
```

- Выполняется ping-проверка доступности
- При неудаче предлагается повторить ввод

### Шаг 3: Пользователь целевого компьютера

```bash
Введите имя пользователя (например, admin или user):
```

- Рекомендуется использовать пользователя с SSH-доступом (часто `admin`)

### Шаг 4: Пароль пользователя

```bash
Введите пароль пользователя <имя>:
```

- Ввод скрыт (не отображается в терминале)
- Выполняется проверка SSH-подключения
- При неудаче можно повторить ввод

### Шаг 5: Целевая папка

```bash
Введите путь к целевой папке на удалённом компьютере:
```

- Проверяется существование папки на удалённом хосте
- Если папки нет — предлагается создать её

### Шаг 6: Настройки копирования

Скрипт спросит о следующих опциях (по умолчанию — включены):

| Опция | Описание | По умолчанию |
| ----- | -------- | ------------ |
| **Архивный режим (-a)** | Сохраняет права, метки времени, ссылки | ✅ да |
| **Подробный вывод (-v)** | Отображает обрабатываемые файлы | ✅ да |
| **Сжатие данных (-z)** | Сжимает данные при передаче | ✅ да |
| **Прогресс (--progress)** | Показывает прогресс копирования | ✅ да |
| **Ускоренное копирование** | Максимальное сжатие + быстрый SSH-шифр | ❌ нет |

### Финал: Подтверждение и копирование

Перед запуском отображается итоговая конфигурация и команда. После подтверждения начинается копирование.

По завершении выводится отчёт:

```bash
✓ Копирование завершено успешно!
Источник:  /home/user/Документы/
Цель:      admin@192.168.1.200:/home/admin/backup/
Время:     5 мин 23 сек
```

## Примеры использования

### Пример 1: Стандартное копирование

```bash
Источник: /home/user/Документы/
Цель:     admin@192.168.1.200:/home/admin/backup/
Опции:    -a -v -z --progress
```

### Пример 2: Ускоренное копирование больших объёмов

```bash
Источник: /home/user/Видео/
Цель:     admin@192.168.1.200:/home/admin/video/
Опции:    -a -v --compress-level=9 --progress (ssh -c aes128-gcm@openssh.com)
```

## Частые проблемы

### 1. Permission denied при SSH-подключении

**Причина:** Обычные пользователи на РЕД ОС могут не иметь SSH-доступа.

**Решение:** Используйте учётную запись администратора (например, `admin`).

### 2. Папка не найдена

**Причина:** Ошибка в пути или регистре имени.

**Решение:** Скрипт предложит похожий вариант. Также можно проверить точное имя:

```bash
ls -la /home/user/
```

### 3. Копирование прервалось

**Решение:** Запустите скрипт снова. Благодаря флагу `--partial` rsync продолжит с места прерывания.

### 4. Недостаточно места на целевом диске

**Решение:** Освободите место на целевом компьютере и запустите скрипт снова.
