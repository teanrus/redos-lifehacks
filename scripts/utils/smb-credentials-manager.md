## 1. Жизненная ситуация

> **Ситуация:** В организации на РЕД ОС настроен доступ к сетевой папке `smb://10.13.60.4/share`. При первом подключении система предложила сохранить пароль, и пользователь нажал «Запомнить навсегда». Через месяц администратор изменил пароль учётной записи для доступа к папке. Теперь при попытке подключения возникает ошибка авторизации — система автоматически использует старый сохранённый пароль, а пользователь не понимает, почему доступ закрыт.
>
> **Проблема:** Где хранится этот пароль и как его удалить? В Linux учётные данные могут быть сохранены в нескольких местах: GNOME Keyring, файлы конфигурации Samba, кэш, файлы монтирования и другие.
>
> **Решение:** Запустите скрипт `smb-credentials-manager.sh` — он найдёт все сохранённые учётные данные и предложит удалить нужные в интерактивном режиме.

```bash
curl -sL https://github.com/teanrus/redos-lifehacks/releases/latest/download/smb-credentials-manager.sh | bash
```

### Способ 2: Скачивание и запуск (рекомендуется)


## 3. Что сканирует скрипт

Скрипт проверяет **12 основных хранилищ** учётных данных:

| № | Хранилище | Путь / Описание | Что ищет |
| - | --------- | --------------- | -------- |
| 1 | **GNOME Keyring** | `secret-tool search` | Записи SMB/CIFS с серверами, пользователями |
| 2 | **Файлы учётных данных Samba** | `~/.smbcredentials`, `/etc/samba/*cred*`, `/etc/fstab` | Логины и пароли для SMB |
| 3 | **Кэш Samba** | `/var/cache/samba`, `~/.cache/samba` | Кэшированные сессии |
| 4 | **Файл ~/.netrc** | `~/.netrc` | Учётные данные для FTP/SMB/HTTP |
| 5 | **GVFS (монтирования)** | `~/.config/gvfs`, `~/.gvfs` | Смонтированные сетевые ресурсы |
| 6 | **GVFS Metadata** | `~/.local/share/gvfs-metadata/` | Метаданные подключений (с именами файлов) |
| 7 | **Конфиг Samba пользователя** | `~/.smb/smb.conf` | Пользовательские настройки SMB |
| 8 | **KDE Wallet** | `~/.local/share/kwalletd/` | Учётные данные KDE (новый) |
| 9 | **Autofs** | `/etc/auto.master`, `/etc/auto.*` | Автоматическое монтирование (новый) |
| 10 | **Systemd mount units** | `/etc/systemd/system/*.mount` | Systemd-монтирование (новый) |
| 11 | **Активные GVFS** | `gio mount --list` | Активные подключения (новый) |
| 12 | **Dconf (GNOME)** | `~/.config/dconf/user` | Настройки GNOME (новый) |

> [!note]
> Для **GVFS Metadata** скрипт теперь показывает **конкретные имена файлов**, чтобы вы могли понять, от какого ресурса хранятся метаданные.

## 4. Режимы работы

### Интерактивный режим (по умолчанию)

```bash
./smb-credentials-manager.sh
```

Скрипт сканирует хранилища, показывает список найденных записей и предлагает удалить их по одной или все сразу.

### Режим только сканирования

```bash
./smb-credentials-manager.sh --scan
```

Только показывает найденные записи без удаления. Полезно для аудита.

### Пакетный режим (удаление всех записей)

```bash
./smb-credentials-manager.sh --batch
```

Автоматически удаляет все найденные записи с подтверждением каждой операции.

### Режим экспорта отчёта

```bash
# Вывод на экран
./smb-credentials-manager.sh --export

# Сохранение в файл
./smb-credentials-manager.sh --export /path/to/report.txt
```

Создаёт подробный отчёт о найденных учётных данных с датой, именем хоста и пользователя.

### Режим диагностики

```bash
./smb-credentials-manager.sh --diagnose
```

Показывает текущее состояние системы: активные монтирования, записи в keyring, кэш, службы.

### Справка

```bash
./smb-credentials-manager.sh --help
```

## 5. Как работает скрипт

### Этапы работы (интерактивный режим)

1. **Сканирование** — проверяет все 12 хранилищ
2. **Отображение** — показывает нумерованный список найденных записей
3. **Интерактивное удаление** — предлагает выбрать запись для удаления
4. **Проверка** — после удаления проверяет, что запись действительно удалена
5. **Повтор** — после удаления показывает обновлённый список
6. **Завершение** — выводит статистику и рекомендации

### Интерактивный режим

После сканирования скрипт отображает все найденные записи и предлагает:

- Ввести **номер записи** для удаления
- Нажать **`a`** для удаления **всех** записей
- Нажать **`q`** для выхода из скрипта
- После удаления список обновляется и можно продолжить или завершить работу

### Дополнительные возможности

- **Проверка удаления**: после удаления записи из GNOME Keyring или GVFS скрипт проверяет, что запись действительно удалена
- **Детальная информация**: для каждой записи показываются сервер, URI и путь к файлу
- **Безопасность**: все операции требуют подтверждения, ничего не удаляется автоматически

## 7. Ручное управление

Если нужно удалить учётные данные вручную, используйте следующие команды:

### Способ 1: GNOME Keyring через Seahorse (графический интерфейс)

> **Рекомендуется для пользователей, не знакомых с терминалом.**

```bash
# Установите Seahorse если не установлен
sudo dnf install seahorse

# Запустите менеджер паролей
seahorse
```

В приложении **Seahorse (Пароли и ключи)**:

1. Откройте раздел **"Пароли"** → **"Вход"** (или "Login")
2. Найдите запись с адресом сетевого ресурса (например, `10.13.60.4` или `smb://...`)
3. Нажмите правой кнопкой → **Удалить**

> [!tip]
> Этот способ удаляет пароль из связки ключей GNOME — именно там хранятся пароли, сохранённые через файловый менеджер при подключении к `smb://`.

### Способ 2: GNOME Keyring через терминал (secret-tool)

```bash
# Поиск записей по серверу
secret-tool search server 10.13.60.4

# Удаление записи
secret-tool clear server 10.13.60.4

# Поиск всех SMB-записей
secret-tool search --all smb
```

> [!note]
> Если `secret-tool` не установлен: `sudo dnf install libsecret-tools`

### Способ 3: Монтирование через /etc/fstab

Проверьте файл `/etc/fstab`:

```bash
cat /etc/fstab
```

Если там есть запись с логином/паролём, отредактируйте:

```bash
sudo nano /etc/fstab
```

Найдите и удалите (или закомментируйте) строки вида:

```ini
//10.13.60.4/share /mnt/share cifs username=admin,password=oldpass,uid=1000,gid=1000 0 0
//10.13.60.4/share /mnt/share cifs credentials=/etc/samba/credentials,uid=1000 0 0
```

После редактирования:

```bash
# Перемонтировать все записи
sudo mount -a

# Или отмонтировать конкретный ресурс
sudo umount /mnt/share
```

### Способ 4: Файл учётных данных (credentials)

Проверьте наличие файлов с учётными данными:

```bash
ls -la ~/.smbcredentials /etc/samba/credentials 2>/dev/null
```

**Пользовательский файл:**

```bash
# Просмотр
cat ~/.smbcredentials

# Содержимое обычно выглядит так:
# username=admin
# password=oldpass

# Удаление
rm ~/.smbcredentials
```

**Системный файл:**

```bash
# Просмотр
sudo cat /etc/samba/credentials

# Удаление
sudo rm /etc/samba/credentials

# Или редактирование (замена пароля)
sudo nano /etc/samba/credentials
```

> [!warning]
> Файлы credentials могут использоваться в `/etc/fstab` — после удаления обновите соответствующие записи.

### Способ 5: Кэш Samba

```bash
# Остановка служб Samba (опционально, но рекомендуется)
sudo systemctl stop smbd nmbd

# Очистка системного кэша
sudo rm -rf /var/cache/samba/*

# Очистка пользовательского кэша
rm -rf ~/.cache/samba/*

# Запуск служб Samba
sudo systemctl start smbd nmbd
```

### Способ 6: Файл ~/.netrc

Файл `~/.netrc` хранит учётные данные для автоматической авторизации (FTP, SMB, HTTP):

```bash
# Просмотр
cat ~/.netrc

# Пример содержимого:
# machine 10.13.60.4 login admin password oldpass

# Удаление записи для конкретного сервера
sed -i '/^machine 10.13.60.4/,+2d' ~/.netrc

# Или полное удаление файла
rm ~/.netrc
```

> [!note]
> Файл `~/.netrc` должен иметь права `600`: `chmod 600 ~/.netrc`

### Способ 7: GVFS (отмонтирование сетевых ресурсов)

GVFS (GNOME Virtual File System) хранит активные подключения:

```bash
# Просмотр подключённых ресурсов
gio mount --list

# Отмонтирование конкретного ресурса
gio mount -u smb://10.13.60.4/share

# Отмонтирование всех SMB-ресурсов
gio mount -a | grep smb | awk '{print $2}' | xargs -I {} gio mount -u {}

# Отмонтирование всех ресурсов
gio mount -a | awk '{print $2}' | xargs -I {} gio mount -u {}
```

> [!tip]
> После отмонтирования GVFS удалит кэшированные учётные данные для этой сессии.

### Способ 8: Метаданные GVFS

```bash
# Просмотр
ls -la ~/.local/share/gvfs-metadata/

# Удаление всех метаданных
rm -rf ~/.local/share/gvfs-metadata/*

# Удаление метаданных для конкретного ресурса
rm ~/.local/share/gvfs-metadata/*10.13.60.4* 2>/dev/null
```

### Способ 9: Конфигурация Samba пользователя

```bash
# Просмотр пользовательского конфига
cat ~/.smb/smb.conf

# Удаление
rm -rf ~/.smb/

# Или редактирование
nano ~/.smb/smb.conf
```

### Способ 10: KDE Wallet (если используется KDE)

Если в РЕД ОС установлена среда KDE, пароли могут храниться в KDE Wallet:

```bash
# Установка менеджера KDE Wallet
sudo dnf install kwalletmanager

# Запуск менеджера
kwalletmanager5
```

В KWalletManager:

1. Откройте кошелек **kdewallet**
2. Найдите записи с `smb://` или адресом сервера
3. Удалите их

### Способ 11: Autofs (автоматическое монтирование)

Если используется autofs для автоматического монтирования:

```bash
# Проверка конфигурации autofs
cat /etc/auto.master
cat /etc/auto.misc  # или другой файл из auto.master

# Удаление записей с учётными данными
sudo nano /etc/auto.misc

# Перезапуск службы
sudo systemctl restart autofs
```

### Способ 12: Systemd mount units

Systemd может управлять монтированием сетевых ресурсов:

```bash
# Поиск unit-файлов для SMB
systemctl list-units --type=mount | grep smb

# Просмотр unit-файла
systemctl cat mnt-share.mount

# Отключение и остановка
sudo systemctl stop mnt-share.mount
sudo systemctl disable mnt-share.mount

# Удаление unit-файла
sudo rm /etc/systemd/system/mnt-share.mount
sudo systemctl daemon-reload
```

## 9. Диагностика проблем

### После удаления пароля

```bash
# Проверьте, что запись удалена из GNOME Keyring
secret-tool search server 10.13.60.4

# Проверьте, что ресурс отмонтирован
mount | grep smb
gio mount --list

# Попробуйте подключиться заново
gio mount smb://10.13.60.4/share
```

### Если пароль не удаляется

| Проблема | Решение |
| -------- | ------- |
| `secret-tool: Невозможно запустить D-Bus` | Запустите из графической сессии (не через SSH) |
| Файл не удаляется | Проверьте права: `ls -la ~/.smbcredentials` |
| GVFS не отмонтируется | Закройте все окна файлового менеджера с этим ресурсом |
| Кэш Samba не очищается | Остановите службу: `sudo systemctl stop smbd nmbd` |

### Проверка подключения после удаления

```bash
# Подключение через терминал
gio mount smb://10.13.60.4/share

# Проверка доступности ресурса
smbclient -L //10.13.60.4 -U username

# Проверка монтирования
df -h | grep share
```

## 🔒 Безопасность

- Скрипт **не отображает пароли** — только метаданные (серверы, имена файлов)
- Для каждого удаления требуется **явное подтверждение**
- Скрипт **не удаляет всё автоматически** — пользователь выбирает каждую запись
- Все операции выполняются **транзакционно** — при ошибке запись остаётся
- **Проверка после удаления** — скрипт подтверждает, что записи действительно удалены
