## Введение

РЕД ОС — российский дистрибутив Linux на базе RPM, разрабатываемый компанией «РЕД СОФТ». Данный документ содержит подробное сравнение версий веток **7.x** и **8.x**, чтобы помочь в выборе и планировании миграции.

### Для кого этот документ

- 🏢 **Системные администраторы** — планирование обновления
- 🔒 **Специалисты по безопасности** — оценка изменений
- 💻 **Разработчики** — сравнение средств разработки
- 📋 **Руководители IT** — планирование инфраструктуры

- ❌ Устаревшие версии пакетов

## Стоит ли обновляться?

### Сценарии обновления

#### ✅ Обновляться на 8.0 если:

- Нужны новые рабочие столы (KDE, GNOME)
- Требуется Wayland
- Нужно современное ПО для разработки
- Нужна лучшая поддержка нового оборудования
- Хотите магазин приложений
- Нужен режим киоска
- Хотите PipeWire вместо PulseAudio
- Требуются последние версии языков программирования

#### ⏸️ Остаться на 7.3 если:

- Всё работает стабильно
- Нет потребности в новых функциях
- Оборудование полностью совместимо
- Требуется максимальная стабильность
- Есть опасения по поводу миграции
- Критичное ПО не сертифицировано для 8.0

#### ❌ Не обновляться если:

- Используются 7.1 или 7.2 (уже EOL — лучше перейти на 7.3 или 8.0)
- Критичная инфраструктура без тестирования
- Нет резервной копии

## Как обновиться с 7.3 до 8.0

### Подготовка

```bash
# 1. Полная резервная копия
sudo tar czf /backup/system-backup-$(date +%Y%m%d).tar.gz \
    /etc /home /var/lib/dnf /var/lib/rpm

# 2. Обновить текущую систему
sudo dnf update -y
sudo reboot

# 3. Проверить статус
cat /etc/redos-release
uname -r
```

### Обновление через redos-majorupgrade

```bash
# 1. Установка утилиты
sudo dnf install -y redos-majorupgrade

# 2. Проверка готовности
sudo redos-majorupgrade check

# 3. Тестовое обновление (без применения)
sudo redos-majorupgrade test

# 4. Применение обновления
sudo redos-majorupgrade run

# 5. Перезагрузка
sudo reboot

# 6. Проверка версии
cat /etc/redos-release
uname -r
```

### После обновления

```bash
# 1. Обновление пакетов
sudo dnf update -y

# 2. Очистка старых пакетов
sudo dnf autoremove

# 3. Проверка служб
systemctl --failed

# 4. Проверка установленного ПО
rpm -qa | sort
```

### Возможные проблемы

| Проблема | Решение |
|----------|---------|
| **Конфликт пакетов** | `sudo dnf distro-sync --allowerasing` |
| **Служба не запускается** | `systemctl status <service>`, проверить конфиг |
| **Графика не работает** | Выбрать другой рабочий стол в GDM/SDDM |
| **Пропало ПО** | Установить заново: `sudo dnf install <package>` |
| **Сеть не работает** | `sudo nmcli connection reload` |
