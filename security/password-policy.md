## 🔐 1. Включите современный алгоритм шифрования
В файле `/etc/login.defs` замените устаревшие `SHA-512` на `yescrypt`:
```bash
ENCRYPT_METHOD yescrypt
YESCRYPT_COST_FACTOR 10
```
>**Зачем:** `yescrypt` устойчивее к брутфорсу, а параметр `cost_factor` замедляет подбор хешей.

## 🛡️ 2. Настройте блокировку после неудачных попыток
В `/etc/pam.d/system-auth` добавьте модуль `pam_faillock`:
```bash
auth required pam_faillock.so preauth silent deny=3 unlock_time=900
auth [default=die] pam_faillock.so authfail deny=3 unlock_time=900
```
>**Зачем:** Аккаунт блокируется на 15 минут после 3 неудачных входов.

## 📏 3. Требуйте сложные пароли через pam_pwquality
В `/etc/security/pwquality.conf` установите:
```conf
minlen = 14
dcredit = -1      # минимум 1 цифра
ucredit = -1      # минимум 1 заглавная
lcredit = -1      # минимум 1 строчная
ocredit = -1      # минимум 1 спецсимвол
retry = 3
```
>Зачем: Пароль должен содержать все типы символов и быть не короче 14 знаков.

## 📜 4. Запретите повторное использование паролей
В `/etc/pam.d/system-auth` добавьте:
```bash
password required pam_pwhistory.so remember=5 use_authtok
```
>**Зачем:** Пользователь не сможет использовать последние 5 своих паролей.

## ⏰ 5. Установите срок действия пароля
```bash
chage --maxdays 90 --mindays 7 --warn 5 username
```
>**Зачем:** Пароль придётся менять каждые 90 дней, но не чаще раза в 7 дней.

## 🔍 6. Проверьте, что хеши используют yescrypt
```bash
grep -E '^\$y\$' /etc/shadow
```
Если видите `$6$` (SHA-512) — пользователи ещё не перешли на новый алгоритм.

## 📊 7. Настройте аудит изменений паролей
В `/etc/audit/rules.d/audit.rules` добавьте:
```bash
-w /usr/bin/passwd -p x -k password_change
-w /etc/shadow -p wa -k shadow_watch
```
>**Зачем:** Будете знать, кто и когда менял пароли или пытался читать /etc/shadow.

## Скрипт для автоматической настройки комплексной политики безопасности паролей в операционной системе **РЕД ОС** (Red OS).

## 📋 Описание

Скрипт предоставляет интерактивный интерфейс для настройки всех основных параметров безопасности паролей:

| № | Компонент | Описание |
|---|-----------|----------|
| 1 | **yescrypt** | Современный алгоритм хеширования с повышенной стойкостью к брутфорсу |
| 2 | **pam_faillock** | Блокировка учётной записи после неудачных попыток входа |
| 3 | **pam_pwquality** | Требования к сложности пароля (длина, типы символов) |
| 4 | **pam_pwhistory** | Запрет на повторное использование старых паролей |
| 5 | **chage** | Срок действия паролей и правила смены |
| 6 | **Права доступа** | Защита системных файлов (`/etc/shadow`, `/etc/passwd`) |
| 7 | **auditd** | Аудит изменений паролей и доступа к чувствительным файлам |
| 8 | **sudo** | Настройка timeout для сессий sudo |

## 📖 Возможности

### ✅ Интерактивный режим
- Выбор пользователей для применения настроек (все или конкретный)
- Пошаговое подтверждение каждого действия
- Настройка минимальной длины пароля (по умолчанию 14 символов)
- Возможность выбрать только нужные пункты меню

### 🔧 Технические возможности

#### 1. Алгоритм хеширования (yescrypt)
```ini
ENCRYPT_METHOD yescrypt
YESCRYPT_COST_FACTOR 10
```
- Устойчивость к GPU-атакам и rainbow tables
- Настраиваемая стоимость вычисления хеша

#### 2. Блокировка после неудачных попыток (pam_faillock)
```ini
deny = 3              # Блокировка после 3 попыток
unlock_time = 900     # Разблокировка через 15 минут
```

#### 3. Сложность паролей (pam_pwquality)
```ini
minlen = 14           # Минимальная длина (настраивается)
dcredit = -1          # Минимум 1 цифра
ucredit = -1          # Минимум 1 заглавная
lcredit = -1          # Минимум 1 строчная
ocredit = -1          # Минимум 1 спецсимвол
minclass = 4          # Минимум 4 класса символов
```

#### 4. История паролей (pam_pwhistory)
```ini
remember = 5          # Запоминание 5 последних паролей
enforce_for_root = 1  # Применять к root
```

#### 5. Срок действия паролей
```bash
--maxdays 90          # Действителен 90 дней
--mindays 7           # Смена не чаще раза в 7 дней
--warn 5              # Предупреждение за 5 дней
--inactive 14         # Блокировка через 14 дней после истечения
```

#### 6. Права доступа к файлам
| Файл | Права | Владелец |
|------|-------|----------|
| `/etc/shadow` | `0000` | `root:root` |
| `/etc/passwd` | `644` | `root:root` |
| `/etc/group` | `644` | `root:root` |

#### 7. Аудит (auditd)
```bash
-w /usr/bin/passwd -p x -k password_change
-w /etc/shadow -p wa -k shadow_access
-w /etc/security/opasswd -p wa -k password_history
-w /etc/pam.d/system-auth -p wa -k pam_config_change
```

#### 8. sudo timeout
```ini
Defaults timestamp_timeout=5
```

## 🔍 Проверка результатов

После выполнения скрипта используйте следующие команды для проверки:

```bash
# Проверка политики паролей пользователя
chage -l $USER

# Проверка алгоритма хеширования (должны начинаться с $y$)
grep -E '^\$y\$' /etc/shadow

# Проверка настройки pam_faillock
grep pam_faillock /etc/pam.d/system-auth

# Проверка настроек pwquality
cat /etc/security/pwquality.conf

# Проверка правил аудита
auditctl -l | grep password
```

## ⚠️ Важные замечания

1. **Резервное копирование**: Скрипт создаёт резервную копию `/etc/pam.d/system-auth` перед внесением изменений.
2. **Смена паролей**: После настройки пользователям потребуется сменить пароли при следующем входе.
2. **Откат изменений**: При необходимости восстановите оригинальный файл из резервной копии:
   ```bash
   cp /etc/pam.d/backup/system-auth.* /etc/pam.d/system-auth
   ```
